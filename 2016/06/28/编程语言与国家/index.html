编程语言与国家

• 标签: 编程语言轶事

来自quora上的一个问题：如果编程语言是国家，那各种语言都代表哪些国家？

Java： 美国——乐观、强大，但喜欢掩饰困难

C++：英国 ——强大并严苛，但实际上不大擅长完成事情，并被 Java 压制了。

Python：荷兰——「嘿，没问题，咱们开干吧！Hey, no problem, let’sh do it guysh!」

Ruby：法国——强大、时尚、相信自己的正确性，但有点被人无视了。

汇编：印度——庞大、不易理解、非常重要，但有很多问题。

COBOL：俄罗斯——曾经非常强大，经理都写在心里了，但最终输了

SQL 和 PL/SQL：德国——坚实、可靠，能挑重担

Javascript:：意大利——很有影响力，深受大家喜欢，但容易掉链子。

Scala：匈牙利——技术纯正、正确，但因痴迷行不通的语法，反而限制了其未来的成功。

C：挪威——强硬、有活力，但不是很刺激。

PHP：巴西——充满美感，比较轻视自己，但背地里却很保守

LISP：冰岛——非常灵活，效率高，不过是一个冰封国度，还很遥远。

Perl：中国——几乎啥都能做，但太神秘了。

Swift：日本—— 1 分钟前还没有，现在到处都是了，并且你手机就依赖它了

C#：瑞士——美丽，深思熟虑，但如果你真想牵扯进来，那就要付出很多。

R：列支敦士登——可能真的很惊人，尤其是你到大数字里面时，但没人知道其实际作用。

Awk：朝鲜——顽固地抗拒变化，其用户貌似不自然地喜欢它，其原因我们也只能猜测了。
